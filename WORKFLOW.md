# 幻灯片 HTML 还原与截图工作流

本文档详细说明了将 `pptx-template` 项目中的幻灯片预览图（PNG 格式）高保真还原为 HTML 代码，并使用 Playwright 进行自动化截图的完整工作流。

## 1. 工作流概述

该工作流的核心目标是实现从“静态图片”到“可编辑网页”的高质量转换。与依赖自动化生成和机器像素对比的方法不同，本工作流强调**人工精细分析**与**高保真手写代码**的结合，以确保还原出的 HTML 页面在字体层级、色块比例、装饰图案和真实图片素材等方面，完全符合“高级商务”模板的设计标准。

整个流程包括以下几个关键阶段：

1.  **原图分析与素材准备**：深入分析原始幻灯片图片的布局结构、色彩搭配（如深宝蓝 `#1c3d7a`）、字体字重（`900` 与 `700` 的对比）以及图文交叠关系。针对原图中的高质量摄影背景，主动搜索并下载同等风格的真实图片素材，以替代简单的 CSS 渐变或色块。
2.  **高保真 HTML 编写**：采用绝对定位（`position: absolute`）进行像素级布局，精确控制文本的字间距（`letter-spacing`）、行高（`line-height`）以及各元素的层级关系（`z-index`）。通过 Python 脚本将下载的图片素材转换为 Base64 编码，直接内嵌至 HTML 的 CSS `background` 属性中，确保网页的独立性与视觉完整性。
3.  **Playwright 自动化截图**：利用无头浏览器环境加载生成的 HTML 文件，并强制设定视口（Viewport）尺寸为 `1280x720`，执行自动化截图，生成与原图尺寸一致的对比图。
4.  **问题记录与迭代优化**：将还原过程中遇到的布局错位、素材缺失或排版粗糙等问题，详细记录至 `records/issues.md` 文档中，并持续迭代优化 HTML 代码，直至截图效果高度贴合原图设计。

## 2. 目录结构说明

为了保证工作流的清晰与可维护性，项目目录按照功能模块进行了合理的划分。

| 目录/文件路径 | 功能描述 |
| :--- | :--- |
| `01高级商务蓝配色/` | 存放原始幻灯片图片及还原产物的核心目录 |
| `01高级商务蓝配色/html/` | 存放手工编写的高保真 HTML 还原文件 |
| `01高级商务蓝配色/html/assets/` | 存放从网络下载并用于 HTML 内嵌的真实摄影图片素材 |
| `01高级商务蓝配色/screenshots/` | 存放由 Playwright 截取的 HTML 页面最终效果图 |
| `scripts/` | 存放支撑工作流的各类 Python 自动化脚本 |
| `scripts/gen_slides_3_4_5.py` | 负责将 Base64 图片注入并批量生成指定幻灯片 HTML 的脚本 |
| `scripts/screenshot_html.py` | 基于 Playwright 的无头浏览器截图脚本 |
| `records/issues.md` | 记录还原实验过程中遇到的问题、根本原因及解决方案 |
| `WORKFLOW.md` | 本文档，提供完整工作流的说明与操作指南 |

## 3. 核心脚本与执行步骤

本节将详细介绍如何运行核心脚本以完成从 HTML 生成到最终截图的整个流程。

### 3.1 环境依赖配置

在执行任何脚本之前，必须确保运行环境已安装必要的 Python 库及浏览器内核。

通过执行以下命令安装依赖：

```bash
sudo pip3 install playwright pillow
python3.11 -m playwright install chromium
```

### 3.2 HTML 文件的生成

HTML 文件的生成不仅涉及代码的编写，还包括将真实图片素材转换为 Base64 编码并注入代码的过程。以 `scripts/gen_slides_3_4_5.py` 为例，该脚本读取 `assets/` 目录下的图片，将其编码后嵌入预先设计好的高保真 HTML 模板中，并最终输出至 `html/` 目录。

执行命令：

```bash
cd /home/ubuntu/pptx-template
python3.11 scripts/gen_slides_3_4_5.py
```

### 3.3 自动化截图生成

在确保 HTML 文件已正确生成后，使用 `scripts/screenshot_html.py` 脚本调用 Playwright 启动无头 Chromium 浏览器，对指定 HTML 页面进行渲染并截图。该脚本会自动将视口设置为 `1280x720`，并将结果保存至 `screenshots/` 目录。

单张截图命令示例：

```bash
python3.11 scripts/screenshot_html.py \
  "01高级商务蓝配色/html/slide_1.html" \
  "01高级商务蓝配色/screenshots/slide_1.png"
```

批量截图可以通过简单的 Shell 循环实现，从而极大地提高了工作效率。

## 4. 最佳实践与注意事项

为了确保还原工作的高质量与流程的顺畅，建议遵循以下最佳实践：

1.  **拒绝单纯依赖机器打分**：像素差异对比（如 SSIM 或 MSE）往往对排版细节（如字重、间距）不敏感。必须通过人工观察原图，准确把握设计意图。
2.  **严格的层级控制**：在复杂的图文交叠场景中，务必在 CSS 中为每一个绝对定位的元素明确分配 `z-index` 值，以防止关键信息（如 LOGO 或主标题）被背景图片或装饰色块遮挡。
3.  **内嵌资源以防失效**：外部图片链接可能因网络原因或源站调整而失效，导致模板展示不完整。将所有必要的视觉素材转换为 Base64 编码并直接内嵌，是保证 HTML 页面稳定性的最有效手段。
4.  **视口尺寸的统一**：无论是 HTML 中的 `<meta name="viewport">` 声明，还是 Playwright 脚本中的 `viewport` 参数设置，都必须严格统一为 `1280x720`，以避免截图出现意外的缩放或截断现象。

通过遵循上述工作流与最佳实践，我们能够高效、高质地完成从静态幻灯片图片到动态 HTML 页面的还原任务。
