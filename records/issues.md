# 实验问题与解决方案记录

> 本文档记录在 PPT 幻灯片 HTML 还原工作流中遇到的问题及对应的解决方案。

---

## 问题记录

### 问题 #1：外部图片素材缺失与还原度低

- **发现时间**：2026-04-07
- **影响范围**：slide_1 ~ slide_5 等包含真实摄影图片的幻灯片
- **问题描述**：原幻灯片模板中大量使用了高质量的商业摄影图片（如仰拍的摩天大楼）。在初期试验中，由于仅使用 CSS 渐变或简单色块模拟，导致生成的 HTML 页面视觉质感与原图相差甚远，无法达到“高级商务”要求。
- **根本原因**：纯代码无法完美复刻复杂的真实摄影纹理与光影。
- **解决方案**：主动通过网络搜索真实素材（如关键词“仰拍 摩天大楼 城市建筑 蓝天”），下载接近原图风格的高清摄影图片至本地 `assets` 目录。在生成 HTML 时，通过 Python 脚本将图片转换为 Base64 编码并内嵌到 CSS 的 `background` 属性中。
- **效果评估**：视觉还原度大幅提升，从简单的色块变成了真实的高级商务背景。
- **状态**：已解决

### 问题 #2：字体层级与排版精细度不足

- **发现时间**：2026-04-07
- **影响范围**：所有包含文本内容的幻灯片
- **问题描述**：自动化生成和机器打分忽略了人类设计模板时对“字体层级”的精细把控。主副标题字重无明显区分，英文字母间距缺失，导致排版粗糙，相似度打分虽不低但视觉效果极差。
- **根本原因**：机器打分算法（如像素差异）对排版细节不敏感，无法代替人类的设计直觉。
- **解决方案**：放弃纯自动化打分依赖，转由人工（Manus）仔细观察原图细节并手写高保真 HTML。严格区分字重（如主标题 `900`，副标题 `700`），调整字间距（`letter-spacing`），并全面采用绝对定位（`position: absolute`）以精确对齐每一个文本块和色块的边界。
- **效果评估**：字体层级分明，排版精细度达到商业模板标准。
- **状态**：已解决

### 问题 #3：复杂图文交叠与层级关系（Z-Index）错误

- **发现时间**：2026-04-07
- **影响范围**：slide_1 等具有多层叠加元素的幻灯片
- **问题描述**：在 slide_1 中，全幅背景图被左侧深蓝色面板遮挡，顶部还有浅灰白色横条，且 LOGO 需要悬浮在横条之上。初期代码未正确处理层级，导致 LOGO 被图片或面板遮挡。
- **根本原因**：绝对定位元素的 `z-index` 属性未明确规划。
- **解决方案**：在编写 CSS 时，为每一个绝对定位元素明确分配 `z-index`。背景图设为 1，顶部横条设为 3，主内容面板设为 4，所有文本内容（LOGO、标题等）设为 6，确保文字始终在最上层可见。
- **效果评估**：图文交叠关系完全正确，无任何遮挡错误。
- **状态**：已解决

### 问题 #4：超大装饰文字尺寸与溢出效果失真

- **发现时间**：2026-04-07
- **影响范围**：slide_6（目录页）、slide_11（章节页）
- **问题描述**：在第一轮还原中，虽然识别出了超大文字（如竖排 `CONTENTS` 和数字 `1`），但字号设置过小，且没有正确处理溢出屏幕和被裁切的视觉效果，导致与原图的视觉冲击力差距很大。
- **根本原因**：对设计中作为“图形装饰”的文字理解不足，将其当作普通文本处理。
- **解决方案**：仔细观察原图元素的边界，如果元素被裁切或超出屏幕，需使用 `position:absolute` 配合负值或超大尺寸（如 `font-size: 580px`）。对于竖排文字，结合 `writing-mode: vertical-rl`、`text-orientation: mixed` 和 `transform: rotate(180deg)` 来实现自下而上的排版，并允许其溢出容器。
- **效果评估**：超大装饰文字的视觉张力和裁切效果完美还原。
- **状态**：已解决

### 问题 #5：背景图片色调与原图氛围不符

- **发现时间**：2026-04-07
- **影响范围**：slide_1、slide_6
- **问题描述**：原图 slide_1 是夜晚暗色调建筑，slide_6 是白天蓝天建筑，但在初次搜索替换图片时没有精确匹配色调，导致整体氛围不对。
- **根本原因**：搜索图片时使用的关键词过于宽泛，未包含环境光线描述。
- **解决方案**：搜索替换图片时，在 query 中加入明确的环境词（如 `dark night`, `blue sky daytime`）。更换为精确匹配氛围的高清无水印摄影图，重新转 Base64 嵌入。
- **效果评估**：图片色调与原图高度一致，整体商务氛围大幅提升。
- **状态**：已解决

### 问题 #6：文本换行导致布局崩塌

- **发现时间**：2026-04-07
- **影响范围**：slide_7 等具有多栏布局的幻灯片
- **问题描述**：在响应式或固定宽度的容器中，长标题（如“目录 | Contents”）容易因为容器宽度不足而意外换行。
- **根本原因**：未限制文本换行行为，且左右面板的宽度分配不合理。
- **解决方案**：对于必须保持单行的标题，强制添加 `white-space: nowrap`。重新计算并调整左右面板的宽度比例（如将左侧面板加宽到 480px），确保文本有足够的显示空间。
- **效果评估**：标题单行显示，布局稳固不崩塌。
- **状态**：已解决
